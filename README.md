# linux-nvidia-edid-fix
Shell script to fix DP1.4 handshake between monitor and NVIDIA GPUs on Linux

> [!WARNING]
> This was AI generated by [OpenCode](https://opencode.ai/) to fix my own issues with my ASUS ROG STRIX XG27AQDMG on Omarchy Linux. Your mileage may vary.

## Usage
* Run the fix: `sudo ./nvidia-dp-edid-fix.sh` 
* Preview only: `sudo ./nvidia-dp-edid-fix.sh --dry-run`
* Revert the fix: `sudo ./nvidia-dp-edid-fix.sh --revert`
* Skip confirmations: `sudo ./nvidia-dp-edid-fix.sh --force`

## How It Works
1. Scans all connected external DRM connectors (DP, HDMI)
2. Detects the NVIDIA dummy EDID by checking for manufacturer NVD and 128-byte fallback
3. If found, looks for a matching connector with a valid EDID (256+ bytes)
4. If no valid EDID is found, guides the user to switch to DP 1.2 mode
5. Copies the real EDID to `/lib/firmware/edid/`
6. Detects bootloader (systemd-boot or GRUB) and backs up + adds `drm.edid_firmware=CONNECTOR:edid/CONNECTOR.bin` to kernel cmdline

## Safety Features
- Boot config backed up before modification
- Dry-run mode to preview
- Revert mode to undo
- Requires root (sudo)
- Validates all dependencies before running
- Skips internal laptop displays (eDP/LVDS/DSI)
